tactic.mk_pattern

list level → list expr → expr → list level → list expr → tactic tactic.pattern

mk_pattern ls es t u o creates a new pattern with (length ls) universe meta-variables and (length es) meta-variables. In the produced pattern p, we have that - pattern.target p is the term t where the universes ls and expressions es have been replaced with temporary meta-variables. - pattern.uoutput p is the list u where the universes ls have been replaced with temporary meta-variables. - pattern.moutput p is the list o where the universes ls and expressions es have been replaced with temporary meta-variables. - pattern.nuvars p = length ls - pattern.nmvars p = length es

The tactic fails if o and the types of es do not contain all universes ls and expressions es.

tactic.match_pattern

tactic.pattern → expr → opt_param tactic.transparency tactic.transparency.reducible → tactic (list level × list expr)

mk_pattern p e m matches (pattern.target p) and e using transparency m. If the matching is successful, then return the instantiation of pattern.output p. The tactic fails if not all (temporary) meta-variables are assigned.

tactic.pexpr_to_pattern

pexpr → tactic tactic.pattern

Given a pre-term of the form λ x_1 ... x_n, t[x_1, ..., x_n], converts it into the pattern t[?x_1, ..., ?x_n]

tactic.match_expr

pexpr → expr → opt_param tactic.transparency tactic.transparency.reducible → tactic (list expr)

Convert pre-term into a pattern and try to match e. Given p of the form λ x_1 ... x_n, t[x_1, ..., x_n], a successful match will produce a list of length n.

tactic.match_subexpr

pexpr → expr → opt_param tactic.transparency tactic.transparency.reducible → tactic (list expr)

Similar to match_expr, but it tries to match a subexpression of e. Remark: the procedure does not go inside binders.
